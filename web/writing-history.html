Writing History for the Web

November 12, 2012

Writing History?

I want to discuss some aspects of html5 for writing a history text. I will discuss aspects like user comments, footnotes, and "chunking" (an ambiguous term that I hope to clarify).

I will not discuss: how to do research, how to structure sentences, what grammar to use, what citation system should be used, what models or historiographical paradigm should be employed, etc. etc. etc.

In principle, the same "tools" presented here could be applied to any field. I just chose history, since I'm reading Gibbon's Decline and Fall.

Chunking

When writing for the web, people don't like to sit down and study long paragraphs. (However, when reading on paper, they love reading paragraphs!)

A different organization scheme is needed. In the 18th and 19th centuries, it seems that historians implicitly used an outline.

"Chunking" is a term I use to describe mathematical writing which numbers each point made (see, e.g., Arthur Besse's writings). Why not use this with an outline structure?

The lay reader in a hurry can read each item in the outline, and get the general idea of what is happening. The scholar can read the body of the chunk and learn about the evidence.

What is an example? Consider the following snippet:

1.0. Overview. "Afghanistan, particularly rural Afghanistan, provides an excellent example of a place where tribal and ethnic groups take primacy over the individual. As a result, any student of Afghan politics must become intimately familiar with such groups and their relationships with one another."

The Social Context of Tribes and Ethnic Groups.

1.1. People's primary loyalty is to their own kin, village, tribe, or ethnic group, generally indicated by the Arabic protean term qawm. [...]

1.2. What is meant by ethnic identity? "Following the work of Fredrik Barth, they are most commonly defined as social groups that meet four criteria: they are biologically replicating, share fundamental cultural values, constitute a field of communication and interaction, and are defined through self-definition and definition by others."

The html5 tags <details> and <summary> works well with the details and the "chunk header" (the bold text in my example).

I would urge the author to use the summary when writing prose, and stick the evidence supporting it (citations, etc.) into the rest of the <details>.

One caveat is that -- as of now -- only Chrome supports <details>. However, there is a jQuery workaround [github.com].

User Comments

I got this idea while reading Real World Haskell, that annotations and user comments should be made on each paragraph.

This requires some technological know-how on the author's part. There are some libraries I can refer to (e.g., stet [gitorious.org]). But the reader wishing to use this should understand how to use a database (e.g., MySQL).

Now, how to do this? It's a bit tricky, since we need to keep track of paragraphs (and we don't want to lose comments or misplace them).

Using [btbytes.com] the Mercurial book's system is easy, but requires quite a few packages (on Ubuntu): 

$ sudo apt-get install docbook
$ sudo apt-get install graphviz
$ sudo apt-get install inkscape
$ sudo apt-get install docbook-xsl docbook-xml docbook-dsssl
$ sudo apt-get install sqlite3
$ sudo apt-get install libsqlite3-dev
$ sudo apt-get install python-virtualenv python-dev

Wooki may be a lazier solution.
